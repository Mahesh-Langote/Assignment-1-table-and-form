Attributes

<a> attributes:

  • href: Specifies the URL of the link
  • target: Specifies where to open the linked document (e.g., "_blank" for a new tab)
    • _blank: Opens the linked document in a new browser window or tab.
    • _self: Opens the linked document in the same frame or window as the link.
    • _parent: Opens the linked document in the parent frame.
    • _top: Opens the linked document in the full body of the window.
    • <frame_name>: Opens the linked document in a named frame.
  • title: Provides a title for the link (often displayed as a tooltip)

    • <img> attributes:

    • src: Specifies the URL of the image
    • alt: Specifies an alternate text for the image (for accessibility)
    • width: Specifies the width of the image
    • height: Specifies the height of the image

    <p> attributes:

    • style: Specifies CSS styles for the paragraph

    <h1> to <h6> attributes:

    • id: Specifies a unique identifier for the heading

    <ol> attributes:

    • type: Specifies the type of the list (e.g., "1" for numeric, "A" for uppercase alphabetical)
      • 1: Numeric numbering (1, 2, 3, ...)
      • A: Uppercase alphabetical (A, B, C, ...)
      • a: Lowercase alphabetical (a, b, c, ...)
      • I: Uppercase Roman numerals (I, II, III, ...)
      • i: Lowercase Roman numerals (i, ii, iii, ...)
    • start: Specifies the start value of the list

    <ul> attributes:

    • type: Specifies the type of list marker
      • square: Square markers
      • circle: Circle markers
      • disc: Disc markers
      • 1: Decimal markers
      • a: Lowercase alphabetical markers
    • start: Not applicable. <ul> elements do not have a start attribute.

    <table> attributes:

    • border: Specifies the border width of the table
    • cellpadding: Specifies the padding between the cell wall and the cell content
    • cellspacing: Specifies the spacing between cells
    • width: Specifies the width of the table
    • height: Specifies the height of the table
    • align: Specifies the alignment of the table (deprecated in HTML5)
    • bgcolor: Specifies the background color of the table (deprecated in HTML5)
    • bordercolor: Specifies the border color of the table (deprecated in HTML5)
    • bordercolordark: Specifies the dark border color of the table (deprecated in HTML5)
    • bordercolorlight: Specifies the light border color of the table (deprecated in HTML5)
    • frame: Specifies which parts of the outside borders that should be visible (deprecated in HTML5)
    • rules: Specifies which parts of the inside borders that should be visible (deprecated in HTML5)

    <input> attributes:

    • type: Specifies the type of input control. Possible values include:
      • text: Text input field
      • password: Password input field
      • checkbox: Checkbox
      • radio: Radio button
      • submit: Submit button
      • reset: Reset button
      • button: Button
      • file: File upload control
      • hidden: Hidden input field
      • image: Image as a submit button
      • date: Date input field
      • number: Number input field
      • email: Email input field
      • url: URL input field
      • tel: Telephone number input field
      • search: Search input field
      • color: Color picker input field
      • accept: Specifies the types of files that the server accepts (for file input). It can take one or more file types or MIME types separated by commas. For example:
        • accept="image/*": Allows only image files
        • accept=".jpg, .jpeg, .png": Allows specific image formats
        • accept="audio/*": Allows only audio files
        • accept=".mp3, .wav": Allows specific audio formats
        • accept=".doc, .docx, .pdf": Allows document files
    • autocomplete: Specifies whether the input field should have autocomplete enabled or disabled
    • autofocus: Specifies that the input field should automatically get focus when the page loads
    • checked: Specifies that a checkbox or radio button should be pre-selected when the page loads
    • disabled: Specifies that the input field should be disabled
    • form: Specifies one or more forms the input field belongs to
    • formaction: Specifies the URL where to submit the form data when the form is submitted (for type="submit" and type="image")
    • formenctype: Specifies how the form data should be encoded before sending it to the server (for type="submit" and type="image")
    • formmethod: Specifies the HTTP method to use when sending form data (for type="submit" and type="image")
    • formnovalidate: Specifies that the form data should not be validated when submitted (for type="submit" and type="image")
    • formtarget: Specifies where to display the response after submitting the form (for type="submit" and type="image")
    • list: Specifies the id of a datalist that contains pre-defined options for an input field
    • max: Specifies the maximum value allowed for an input field (for type="number" or type="date")
    • maxlength: Specifies the maximum number of characters allowed in an input field
    • min: Specifies the minimum value allowed for an input field (for type="number" or type="date")
    • multiple: Specifies that the user is allowed to enter multiple values in an input field (for type="email" or type="file")
    • name: Specifies the name of the input field, used for form submission and identifying the field in scripting
    • pattern: Specifies a regular expression pattern that the input value must match (for type="text" or type="password")
    • placeholder: Specifies a short hint that describes the expected value of the input field
    • readonly: Specifies that the input field is read-only and cannot be edited
    • required: Specifies that the input field is required and must be filled out before submitting the form
    • size: Specifies the width of the input field, in characters (for type="text" or type="password")
    • src: Specifies the URL of an image to display in the input field (for type="image")
    • step: Specifies the legal number intervals for an input field (for type="number")value: Specifies the initial value of the input field

    <button> attributes:

    • type: Specifies the type of the button. Possible values include:
      • submit: Submits the form data
      • reset: Resets the form data to its initial values
      • button: Standard button (default behavior)
    • name: Specifies the name of the button, used for form submission and identifying the button in scripting
    • value: Specifies the value associated with the button, which is submitted with the form data
    • form: Specifies one or more forms the button belongs to
    • formaction: Specifies the URL where to submit the form data when the button is clicked (overrides the form's action attribute)
    • formenctype: Specifies how the form data should be encoded before sending it to the server (for formaction)
    • formmethod: Specifies the HTTP method to use when sending form data (for formaction)
    • formnovalidate: Specifies that the form data should not be validated when submitted (for formaction)
    • formtarget: Specifies where to display the response after submitting the form (for formaction)
    • disabled: Specifies that the button should be disabled
    • autofocus: Specifies that the button should automatically get focus when the page loads
    • onclick: Specifies a JavaScript function to execute when the button is clicked

    <div> attributes:

    • class: Specifies one or more class names for an element (used for styling with CSS)

    <span> attributes:

    • title: Specifies extra information about an element (often displayed as a tooltip)

    <form> attributes:

    • action: Specifies the URL where the form data will be sent
    • method: Specifies the HTTP method to use when sending form data (e.g., "get" or "post")
    • autocomplete: Specifies whether form elements should have autocomplete enabled
    • novalidate: Specifies that a form should not be validated when submitted
    • enctype: Specifies how form data should be encoded before sending it to the server
    • target: Specifies where to display the response after submitting the form
    • name: Specifies a name for the form
    • id: Specifies a unique id for the form

    <label> attributes:

    • for: Specifies which form element a label is bound to (by matching the 'for' attribute with the 'id' attribute of the form element)

    <select> attributes:

    • name: Specifies the name of the select element (used for form submission)
    • required: Specifies that the user is required to select a value before submitting the form

    <option> attributes:

    • value: Specifies the value to be sent to the server when the form is submitted

    <textarea> attributes:

    • name: Specifies the name of the textarea (used for form submission)
    • placeholder: Specifies a short hint that describes the expected value of the textarea
    • rows: Specifies the visible height of the textarea (in lines)
    • cols: Specifies the visible width of the textarea (in characters)

    <audio> attributes:

    • src: Specifies the URL of the audio file
    • controls: Specifies that audio controls should be displayed (e.g., play, pause, volume)

    <video> attributes:

    • src: Specifies the URL of the video file
    • controls: Specifies that video controls should be displayed (e.g., play, pause, volume)

    <iframe> attributes:

    • src: Specifies the URL of the page to embed in the iframe
    • width: Specifies the width of the iframe
    • height: Specifies the height of the iframe

    <script> attributes:

    • src: Specifies the URL of the external script file
    • type: Specifies the scripting language of the embedded code

    <style> attributes:

    • type: Specifies the style sheet language of the embedded styles

    <link> attributes:

    • href: Specifies the URL of the external resource being linked
    • rel: Specifies the relationship between the current document and the linked document
    • type: Specifies the media type of the linked document

    <footer> attributes:

    • class: Specifies one or more class names for the element (used for styling with CSS)

    <header> attributes:

    • class: Specifies one or more class names for the element (used for styling with CSS)

    <nav> attributes:

    • class: Specifies one or more class names for the element (used for styling with CSS)

    <section> attributes:

    • class: Specifies one or more class names for the element (used for styling with CSS)

    <aside> attributes:

    • class: Specifies one or more class names for the element (used for styling with CSS)

    Special Symbols

    < - Less than: &lt;

    > - Greater than: &gt;

    & - Ampersand: &amp;

    " - Double quote: &quot;

    ' - Single quote or apostrophe: &apos; or &#39;

      - Non-breaking space: &nbsp;

    © - Copyright symbol: &copy;

    ® - Registered trademark symbol: &reg;

    ™ - Trademark symbol: &trade;

    × - Multiplication symbol: &times;

    ÷ - Division symbol: &divide;

    # - Number sign or pound sign: &num; or &#35;

    $ - Dollar sign: &dollar;

    % - Percent sign: &percnt;

    † - Dagger symbol: &dagger;

    ‡ - Double dagger symbol: &Dagger;

    • - Bullet point: &bull;

    … - Ellipsis: &hellip;

    – - En dash: &ndash;

    — - Em dash: &mdash;

    ‘ - Left single quotation mark: &lsquo;

    ’ - Right single quotation mark: &rsquo;

    “ - Left double quotation mark: &ldquo;

    ” - Right double quotation mark: &rdquo;

    « - Left-pointing double angle quotation mark: &laquo;

    » - Right-pointing double angle quotation mark: &raquo;

    ‹ - Left-pointing single angle quotation mark: &lsaquo;

    › - Right-pointing single angle quotation mark: &rsaquo;
